# Nmap Network Security Lab

## Overview

This project demonstrates how Nmap can be used to perform basic network reconnaissance and service discovery in an authorized lab environment. The goal of the lab is to understand how open ports and service versions can reveal useful security information about a host.

Nmap is widely used by both network administrators and cybersecurity professionals to identify active systems, discover exposed services, and assess basic network risk.

## Objectives

- Learn how host discovery works
- Identify open TCP ports on a target system
- Detect running services and versions
- Interpret scan results from a security perspective
- Practice documenting findings in a structured format

## Tools Used

- Nmap
- Linux terminal
- Basic TCP/IP networking knowledge

## Skills Demonstrated

- Network reconnaissance
- Port scanning
- Service version detection
- Risk identification
- Security documentation

## Files in This Repository

- `scan-results.md` - Example scan output and observations
- `common-nmap-commands.md` - Frequently used Nmap commands
- `security-analysis.md` - Security interpretation of scan findings



## What I Learned

This lab strengthened my understanding of how services are exposed on a network, how attackers and defenders gather network information, and how scan results can be used to improve security posture.
